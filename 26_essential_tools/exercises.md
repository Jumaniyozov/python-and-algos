# Essential Tools - Exercises

## 15 Practical Exercises

### Exercise 1: Rich Dashboard
**Difficulty**: Easy

Create a system dashboard using Rich that displays:
- Current time and date
- System information (CPU, memory usage)
- A table of running processes
- Color-coded status indicators

Requirements:
- Use Rich Console, Table, and Panel
- Update display every 5 seconds
- Use colors: green for normal, yellow for warning, red for critical

## Exercise Guidelines

### Testing Your Solutions

1. **Functionality**: Does it work as specified?
2. **Error Handling**: Handles invalid input gracefully?
3. **User Experience**: Clear output and helpful messages?
4. **Code Quality**: Clean, readable, well-organized?
5. **Logging**: Appropriate log levels and messages?
6. **Validation**: Robust input validation with Pydantic?

### Evaluation Criteria

| Criteria | Weight | Description |
|----------|--------|-------------|
| Correctness | 30% | Meets all requirements |
| Code Quality | 25% | Clean, organized, documented |
| Error Handling | 20% | Handles edge cases |
| UX | 15% | User-friendly output |
| Validation | 10% | Proper input validation |

### Tips

1. Start with basic functionality, then add features
2. Test with valid and invalid inputs
3. Use type hints consistently
4. Add docstrings to functions
5. Handle errors gracefully with helpful messages
6. Use Rich to make output beautiful and informative
7. Log important events and errors
8. Validate all user input with Pydantic
