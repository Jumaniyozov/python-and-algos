## Common Mistakes to Avoid

### Technical Mistakes

1. **Not handling edge cases**
   ```python
   # âŒ Missing checks
   def solution(arr):
       return arr[0]  # Crashes on empty array!

   # âœ… Handle edge cases
   def solution(arr):
       if not arr:
           return None
       return arr[0]
   ```

2. **Off-by-one errors**
   ```python
   # âŒ Wrong range
   for i in range(len(arr)):
       if i < len(arr):  # Redundant check
           arr[i] = arr[i+1]  # Index error at i=len(arr)-1!

   # âœ… Correct range
   for i in range(len(arr) - 1):
       arr[i] = arr[i+1]
   ```

3. **Modifying while iterating**
   ```python
   # âŒ Modifying during iteration
   for item in list:
       if condition:
           list.remove(item)  # Skips elements!

   # âœ… Iterate over copy or use list comprehension
   list = [item for item in list if not condition]
   ```

4. **Not considering time complexity**
   ```python
   # âŒ Nested loops on large input
   for i in range(n):
       for j in range(n):  # O(nÂ²) - might timeout!
           process(i, j)

   # âœ… Optimize with hash map
   seen = {}
   for i in range(n):
       if complement in seen:  # O(n)
           return [seen[complement], i]
   ```

## Communication Strategies

### The Perfect Interview Flow

**Phase 1: Understanding (5 min)**
```
You: "Let me make sure I understand the problem.
     [Restate problem]
     Can I ask a few clarifying questions?
     [Ask 3-5 questions]
     Great, so to confirm: [restate with answers]"

Interviewer: "Yes, that's correct."

You: "Perfect, let me think about approaches..."
```

**Phase 2: Planning (5-7 min)**
```
You: "I can think of two approaches:

     Approach 1: [Brute force]
     - Time: O(nÂ²), Space: O(1)
     - Simple but slow for large inputs

     Approach 2: [Optimized]
     - Time: O(n), Space: O(n)
     - Uses hash map for faster lookup

     I think Approach 2 is better because [reason].
     Let me outline the algorithm:

     [Outline steps]

     Let me trace through an example:
     [Walk through]

     Time: O(n), Space: O(n)

     Does this sound good?"

Interviewer: "Yes, go ahead."
```

**Phase 3: Implementing (15-20 min)**
```
You: "I'll start coding now.
     [Explain while coding]
     I'm creating a hash map here to...
     This loop goes through...
     Here I'm checking if..."

[Writing code with clear structure and comments]
```

**Phase 4: Testing (5 min)**
```
You: "Let me test this with the example:
     [Trace through step by step]

     Let me also check edge cases:
     - Empty input: [test]
     - Single element: [test]
     - All same: [test]

     Everything looks good!"
```

**Phase 5: Complexity Analysis (3 min)**
```
You: "Time complexity: O(n) because [explain]
     Space complexity: O(n) because [explain]

     We're trading space for time here, which is
     optimal for this problem."

Interviewer: "Great! Any optimizations?"

You: "For this specific problem, we can't do better
     than O(n) time since we need to examine all elements.
     If space was critical, we could [alternative],
     but it would be O(nÂ²) time."
```

```
0:15 - 0:30  Implementation
0:30 - 0:35  Testing & Debugging
0:35 - 0:40  Complexity Analysis
0:40 - 0:45  Follow-up Questions & Discussion
```

### Red Flags (Time Warning Signs)

```
âš ï¸ 0:10 - Still clarifying
   â†’ Move to planning quickly

âš ï¸ 0:20 - No code written
   â†’ Start implementing, refine while coding

âš ï¸ 0:35 - Still coding
   â†’ Write pseudocode for remaining parts
   â†’ Explain what you would implement

âš ï¸ 0:40 - Haven't tested
   â†’ Quick trace through one example
   â†’ Mention edge cases verbally
```

### Time-Saving Techniques

1. **Use templates**
   - Have standard patterns memorized
   - DFS, BFS, binary search templates
   - Reduces implementation time

2. **Write modular code**
   - Use helper functions
   - "I'll use a helper function here..."
   - Explain logic, implement later if needed

3. **Prioritize correctness**
   - Better to have correct slow solution
   - Than fast buggy solution
   - Can optimize after it works
